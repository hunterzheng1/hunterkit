## 2. 接口与通信规范

### 2.1 统一返回体

**成功响应**：
```json
{
  "code": 0,
  "msg": "success",
  "data": { /* 业务数据 */ }
}
```

**失败响应**：
```json
{
  "code": -1,
  "msg": "错误描述",
  "data": null
}
```

### 2.2 分页请求/响应标准

**请求参数**：
| 参数 | 类型 | 必填 | 默认值 | 说明 |
|-----|------|------|-------|------|
| page | 数字 | 否 | 1 | 页码（从1开始） |
| pageSize | 数字 | 否 | 20 | 每页条数（最大100） |

**响应结构**：
```json
{
  "code": 0,
  "msg": "success",
  "data": {
    "list": [],
    "total": 100,
    "page": 1,
    "pageSize": 20
  }
}
```

### 2.3 错误码规范

| 错误码范围 | 含义 |
|-----------|------|
| 0 | 成功 |
| 1000-1999 | 参数校验错误 |
| 2000-2999 | 业务逻辑错误 |
| 3000-3999 | 权限错误 |
| 4000-4999 | 外部服务错误 |
| 5000-5999 | 系统内部错误 |
