# spec.md - 能力规格定义（增量）

> **定位**：`harness-inspect` 的实施偏移修复规格
> **【质量红线】严禁描述模糊；约束必须量化
> **【格式要求】** 需求项使用 `####`（4个#），场景必须使用 `#####`（5个#）

## 1. 需求规格（官方格式）

### 新增需求

无。

### 修改需求

#### 需求项：`--full` 全量扫描参数

系统必须实现 `--full` 参数，支持首次接入或大变更后的全量扫描。

##### 场景：全量扫描
- **当** 用户执行 `harness inspect --full`
- **预期** 系统必须扫描项目全部构建文件、源码目录、文档目录、AI agent 文件、CI workflow 和已知 Harness/旧来源目录，并生成完整的 `.harness/facts/repo-map.json`

##### 场景：首次无 facts 等价全量
- **当** 用户执行 `harness inspect`（无 `--full`）且 `.harness/facts/repo-map.json` 不存在
- **预期** 系统必须自动等价于 `--full` 执行全量扫描

#### 需求项：`--path` 限定扫描参数

系统必须实现 `--path` 参数，支持限定扫描目录。

##### 场景：限定路径扫描
- **当** 用户执行 `harness inspect --path src/payment --json`
- **预期** 系统必须只扫描 `src/payment` 及其必要上下文，在 JSON 中标记 `scope.path` 为 `src/payment`，且禁止越界读取项目根目录外路径

##### 场景：路径不存在
- **当** 用户执行 `harness inspect --path nonexistent`
- **预期** 系统必须返回错误码 2301 并提示路径不存在

#### 需求项：`--rules` 条件生成参数

系统必须实现 `--rules` 参数，支持生成规则建议文件。

##### 场景：生成规则建议
- **当** 用户执行 `harness inspect --rules`
- **预期** 系统必须写入 `.harness/generated/rules.generated.md`，并将不确定事实标记为 `REVIEW_REQUIRED`

##### 场景：规则建议内容
- **当** `--rules` 生成完成
- **预期** 规则文件必须包含模块名、路径、主要语言、构建入口、依赖关系和 agent 规则候选

### 移除需求

无。

## 2. 技术契约（SDD 扩展）

### 2.1 接口定义

#### 接口基本信息
- **路径**：`CLI: harness inspect`
- **方法**：本地进程调用
- **内容类型**：终端文本；`--json` 时为 `application/json`

#### 请求参数

| 参数名 | 类型 | 必填 | 说明 | 示例值 | 约束条件 |
|-------|------|------|------|--------|----------|
| --full | boolean | 否 | 全量扫描 | `true` | 默认 `false`；首次无 facts 时等价于 `true` |
| --path | path | 否 | 限定扫描目录 | `src/payment` | 必须位于 `--cwd` 内 |
| --rules | boolean | 否 | 生成规则建议 | `true` | 为 `true` 时写入 `rules.generated.md` |
| --json | boolean | 否 | JSON 输出 | `true` | stdout 必须是合法 JSON |

#### 错误码定义
| 错误码 | 含义 | 触发条件 |
|-------|------|----------|
| 2301 | 扫描目标不存在 | `--cwd` 或 `--path` 不存在 |
| 2302 | 路径越界 | `--path` 解析后不在项目根目录内 |
| 2303 | facts 写入失败 | `.harness/facts/repo-map.json` 写入失败 |
| 2304 | 规则生成失败 | `--rules` 输出失败 |

## 3. 物理约束

### 3.1 性能约束
| 指标 | 约束值 | 说明 |
|------|-------|------|
| 全量扫描时间 | < 5000 毫秒 (P95) | 文件数 < 5000 |
| 限定路径扫描时间 | < 3000 毫秒 (P95) | 文件数 < 1000 |
| facts JSON 大小 | < 5 MB | 超出时摘要化 |

### 3.2 资源约束
| 资源 | 限制 | 说明 |
|------|------|------|
| 内存 | < 512 MB | 全量扫描 |

### 3.3 超时配置
- 总超时：180000 毫秒
