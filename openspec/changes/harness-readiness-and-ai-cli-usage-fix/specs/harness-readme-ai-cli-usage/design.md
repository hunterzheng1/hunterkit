### 6.1 核心流程 — README 重构步骤

```
[1. 章节重排] 
  → 拆分为 "使用方式（目标项目用户）"（第 1 非标题章节）
  → "开发（本仓库贡献者）"（独立章节，末尾）
  
[2. 用户路径替换]
  → "使用方式" 章节：

    - 第 2 段：Claude Code 使用说明（对话中输入 npx...）
    - 第 3 段：Codex 使用说明（对话中输入 npx...）
    - 第 4 段：初始化后使用（后续命令由 AI 工具 CLI 触发）
  → 移除原"安装"章中的 `npm install` 和 `npm run build`

[3. 命令表格增强]
  → 在原有命令表格增加 "状态" 列
  → 标注每个命令的 M1 状态：✅ 已可用 / 🔶 受限 / ⏳ 后续版本

[4. 开发者章节]
  → 从原"安装""开发"章内容移入
  → 顶部添加警告："以下内容仅适用于本仓库开发者"
  → 保留 npm install、npm run build、npm test 等开发命令

[5. 内部名称审计]
  → 全文搜索：docsync、gsd、kld-sdd、kld-review
  → 若出现则替换为对应对外命令名或删除
  → 确保命令列表、help 文本、示例代码中无内部来源名
```
