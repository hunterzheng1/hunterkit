## 6. 模块内部逻辑

### 6.1 核心流程 — 标注和路径修正

```
[1. Hook 脚本标注]
  → 每个 .sh 脚本头部增加注释块：
    - "Generated by @hunterzheng/harness — 模板脚本"
    - 触发时机说明
    - 状态标注

[2. Hook 配置文件标注]
  → settings.json / hooks.json 增加：
    - _generated_by: "@hunterzheng/harness"
    - _generated_at: ISO 时间戳

[3. Subagent 模板标注]
  → 每个 agent 定义文件的 frontmatter/yaml header 增加：
    - _status: "template"
    - _note: "本 agent 为模板定义..."

[4. 事件路径统一]
  → 新事件写入 .harness/events/<timestamp>-<type>.json
  → 旧 skywalk-sdd/ 事件保持只读兼容
  → session-summary hook 写入目标从旧路径改为新路径

[5. status/doctor 展示]
  → status 命令输出 safety 能力矩阵
  → doctor 命令诊断 Hook 时标注 "模板脚本，完整调度后续版本"
```
