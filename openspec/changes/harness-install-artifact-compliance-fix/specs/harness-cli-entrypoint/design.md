# 局部技术实现方案 - harness-cli-entrypoint

> **定位**：单一 Capability 的业务维度技术实现方案
>
> **边界声明**：本设计仅服务于 `harness-cli-entrypoint`，聚焦无参数 npx 入口、交互式向导选择回显、AI 工具 CLI 口径、安装摘要与 artifact 分类。Skill/Hook/doctor/sync 的具体生成与诊断逻辑由对应 Capability 实现，本能力只负责把选择、摘要和分类结果准确呈现给用户。
>
> **质量红线**：用户只需要记住 `npx @hunterzheng/harness`；所有安装细节由向导收集；安装完成摘要必须让用户看清“选了什么、写了什么、跳过了什么”。

## 4. 局部后端接口设计

### 4.1 接口清单

| 接口名称 | 路径 | 方法 | 说明 |
|---------|------|------|------|
| 无参数初始化入口 | `CLI: npx @hunterzheng/harness` | 本地进程调用 | 首次运行进入初始化向导，已初始化进入菜单 |
| 向导执行 | `internal: runInitWizard(context)` | 内部函数 | 收集项目路径、AI 工具、能力、项目类型、写入策略、Hook 强度 |
| AI CLI 上下文检测 | `internal: detectAiCliContext(env)` | 内部函数 | 识别 Claude Code / Codex / unknown |
| 安装摘要构建 | `internal: buildInstallSummary(answers, artifacts, skipped)` | 内部函数 | 输出 summary 六类信息 |
| CLI 响应输出 | `internal: formatHumanSummary(response)` | 内部函数 | human 模式展示结构化摘要 |

### 4.2 接口详细设计

#### 接口 1：无参数初始化入口

**基本信息**：
- 路径：`CLI: npx @hunterzheng/harness`
- 方法：本地进程调用
- 认证：不需要

**请求参数**：
| 参数名 | 类型 | 必填 | 说明 | 约束 |
|-------|------|------|------|------|
| `argv` | string[] | 是 | CLI 参数数组 | 长度为 0 时进入 interactive |
| `env` | `NodeJS.ProcessEnv` | 是 | AI CLI 环境变量来源 | 只读 |
| `cwd` | path string | 否 | 目标项目根目录 | 默认 `process.cwd()`，必须存在 |

**响应结构**：
```json
{
  "code": 0,
  "msg": "success",
  "data": {
    "command": "init",
    "mode": "wizard",
    "installSummary": {
      "selectedAiTools": ["Claude Code"],
      "selectedCapabilities": ["inspect", "sync"],
      "hookStrength": "full",
      "writeStrategy": "write",
      "runtimeProjectionsWritten": [".claude/skills/harness/SKILL.md"],
      "runtimeProjectionsSkipped": [
        { "tool": "codex", "path": ".agents/skills/harness/SKILL.md", "reason": "tool not selected" }
      ]
    }
  },
  "warnings": [],
  "artifacts": []
}
```

**业务逻辑**：
1. `main(argv, env, io)` 将 `env` 注入 `CommandContext` 或局部 `AiCliContext`。
2. `argv.length === 0` 且无已初始化配置时进入 `runInitWizard()`。
3. wizard 成功后执行 post-wizard integration。
4. post-wizard integration 将 workspace/config/source/runtime/skipped artifacts 分类并构建 install summary。
5. `writeCliResponse()` 在 JSON 模式输出完整结构，在 human 模式渲染六类摘要。

#### 接口 2：向导执行

**基本信息**：
- 路径：`internal: runInitWizard(context)`
- 方法：内部函数
- 认证：不需要

**请求参数**：
| 参数名 | 类型 | 必填 | 说明 | 约束 |
|-------|------|------|------|------|
| `context.globalOptions.cwd` | path string | 是 | 默认目标项目根目录 | 必须存在 |
| `context.aiCliContext.tool` | string | 否 | 当前 AI CLI 工具名称 | `Claude Code`、`Codex`、`Unknown` |
| `context.io` | CliIo | 是 | CLI 输入输出 | 不在 JSON 模式写额外提示到 stdout |

**响应结构**：
```json
{
  "code": 1010,
  "msg": "No AI tool selected",
  "data": {
    "command": "init",
    "mode": "wizard",
    "suggestion": "请选择至少一个 AI 工具"
  },
  "warnings": []
}
```

**业务逻辑**：
1. wizard 首屏 message 增加 AI CLI 上下文，例如 `当前由 Claude Code 触发`。
2. AI 工具 checkbox 返回空数组时，立即返回 `1010`，不继续后续步骤。
3. 成功选择后，将 `selectedAiToolLabels` 写入 `response.data.wizardAnswersSummary`。
4. 已初始化菜单使用同一 `AiCliContext` 在 select message 中提示当前上下文。

#### 接口 3：安装摘要输出

**基本信息**：
- 路径：`internal: formatHumanSummary(response)`
- 方法：内部函数
- 认证：不需要

**请求参数**：
| 参数名 | 类型 | 必填 | 说明 | 约束 |
|-------|------|------|------|------|
| `response.data.installSummary` | InstallSummary | 否 | 安装摘要 | 存在时 human summary 必须展示六类信息 |
| `response.artifacts[]` | CliArtifact[] | 否 | 产物列表 | 每项 type 必须属于规范枚举或映射到规范枚举 |

**响应结构（human）**：
```text
[OK] success
  Command: init
  Selected AI tools: Claude Code
  Selected capabilities: inspect, sync, develop, review, knowledge
  Hook strength: full
  Write strategy: write
  Runtime projections written:
    - [runtime] .claude/skills/harness/SKILL.md
  Runtime projections skipped:
    - [runtime] .agents/skills/harness/SKILL.md (tool not selected)
  Artifacts:
    - [workspace] .harness
```

**业务逻辑**：
1. 如果 `installSummary` 存在，优先输出 summary 六类信息。
2. `Artifacts` 继续输出完整产物列表，但不得把 `.harness/adapters/**` 标为已生效 runtime。
3. JSON 模式不做 human 拼接，保持纯 JSON。

## 6. 模块内部逻辑

### 6.1 核心流程

```text
[main(argv=[], env)]
  --> [parseGlobalOptions]
  --> [buildAiCliContext(env)]
  --> [runInteractiveEntrypoint(context)]
  --> [runInitWizard(context)]
  --> [validate aiTools non-empty]
  --> [executePostWizardIntegration]
  --> [classify artifacts]
  --> [buildInstallSummary]
  --> [writeCliResponse]
```

### 6.2 状态机

```text
[not initialized] --no command--> [init wizard]
[initialized] --no command--> [operation menu]
[wizard step aiTools] --empty selection--> [error 1010]
[wizard complete] --write strategy preview/dry-run--> [summary with zero writes]
[wizard complete] --write strategy write--> [post integration + summary]
[tool selected] --runtime generated--> [runtimeProjectionsWritten]
[tool not selected] --runtime applicable--> [runtimeProjectionsSkipped]
```

### 6.3 关键算法

#### 6.3.1 AI CLI 上下文检测

1. 新增 `detectAiCliContext(env: NodeJS.ProcessEnv): AiCliContext`。
2. 检测顺序：
   - `CLAUDE_CODE_SESSION_ID` 存在：`tool = "Claude Code"`。
   - `CODEX_SESSION_ID` 或 Codex 桌面/CLI 可识别环境变量存在：`tool = "Codex"`。
   - 否则：`tool = "Unknown"`。
3. `interactive.ts` 不再直接读取 `process.env`，所有测试通过传入 env 构造上下文。

#### 6.3.2 AI 工具选择校验

1. checkbox 返回 `aiTools` 后立即检查长度。
2. 如果长度为 0，返回：
   - `code: 1010`
   - `msg: "No AI tool selected"`
   - `data.suggestion: "请选择至少一个 AI 工具"`
3. 不再继续能力、项目类型、写入策略、Hook 强度步骤。
4. 更新旧测试“skip defaults to claude”为新失败用例。

#### 6.3.3 Artifact 分类

1. 将 post-wizard 产物来源分为：
   - `.harness` 目录：`workspace`
   - `.harness/config/**`：`config`
   - `.harness/adapters/**`：`source`
   - `.claude/**`、`.agents/**`、`.codex/**`：`runtime`
   - `.harness/reports/**`：`report`
2. selected tools 对应 runtime 进入 `runtimeProjectionsWritten`。
3. unselected tools 的 runtime 目标进入 `runtimeProjectionsSkipped`，reason 固定为 `tool not selected`。
4. `.harness/adapters/**` 只能显示为 source，不能显示为 runtime。

#### 6.3.4 Human summary 渲染

1. `formatHumanSummary()` 检测 `response.data.installSummary`。
2. 按固定顺序输出：
   - Selected AI tools
   - Selected capabilities
   - Hook strength
   - Write strategy
   - Runtime projections written
   - Runtime projections skipped
3. 如果某列表为空，输出 `none`，避免空白行造成误解。
4. 保持 JSON 模式只输出 `JSON.stringify(response)`。

## 8. 异常处理

### 8.1 异常分类

| 异常类型 | 触发条件 | 处理策略 | 用户感知 |
|---------|---------|---------|---------|
| 参数校验异常 | `--cwd` 不存在 | 保持现有 `1002` | 输出路径不存在 |
| 向导输入异常 | AI 工具空选 | 返回 `1010`，不默认 Claude | 提示至少选择一个 AI 工具 |
| 安装摘要异常 | post-wizard 后缺少六类摘要字段 | 返回或 warning `1011`，JSON 中保留原始 artifacts | 提示安装摘要不完整 |
| help 输出异常 | help 渲染失败 | 保持/补充 `1005` | 提示 help 输出失败 |
| post-wizard 集成异常 | workspace 或 artifact 写入失败 | 保持 warning 聚合，不吞掉已完成摘要 | 用户看到 warning 和已写产物 |
| unknown AI CLI | 无识别 env | 不阻断，显示 `Unknown` | 向导仍可继续 |

### 8.2 重试与降级

- 重试次数：0；CLI 交互错误不自动重试，由用户重新运行。
- 重试间隔：N/A。
- 降级策略：
  - AI CLI unknown：继续向导，但提示当前上下文未知。
  - dry-run/preview：不写文件，仍生成 summary，runtime written 为 `none` 或 planned 状态。
  - post-wizard warning：保留 warning，summary 中展示已完成和跳过项。
