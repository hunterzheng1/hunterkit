## 1. 需求规格（官方格式）

### 新增需求

#### 需求项：safety 功能 M1 完成定义

系统必须在 safety 功能中明确 M1 阶段的完成边界，避免未完成功能冒充已实现。

##### 场景：M1 已实现功能
- **当** 用户执行 safety 相关功能
- **预期** M1 阶段已实现：5 个 Hook 脚本的模板文件生成（`.--harness/adapters/<tool>/hooks/*.sh`）、Hook 配置文件生成（Claude `settings.json`、Codex `hooks.json`）、Subagent 定义文件模板生成（19 个 agent 定义文件）、CLI 命令级危险命令阻断（在 `main.ts` 拦截点）、阻断列表（6 个危险命令）

##### 场景：M1 受限功能
- **当** 用户触发受限制的安全功能
- **预期** M1 阶段受限：Hook dispatcher 未完成（AI 工具 CLI 侧 hook 代理和事件路由框架）、事件审计闭环未完成（`.harness/events/` 写入和完整性验证）、push gate 未完成（Git push 前置检查未集成到 AI 工具 hook 链）、subagent 写入冲突编排未完成（多 agent 并行写入冲突检测和合并策略）

##### 场景：受限功能标注
- **当** 用户访问 Hook 相关功能
- **预期** 系统必须在 `status` 和 `doctor` 命令中展示 safety 各功能的完成状态（已实现/受限/后续版本），受限功能必须附带说明"后续版本支持"

#### 需求项：Hook 脚本模板标注

系统必须确保生成的 Hook 脚本标注其模板性质，引导用户根据项目需求自定义。

##### 场景：Hook 脚本头部注释
- **当** 系统生成 `.sh` Hook 脚本
- **预期** 每个脚本头部必须包含注释标注"Generated by @hunterzheng/harness — 模板脚本，请根据项目需求修改"，并说明脚本的触发时机和预期行为

##### 场景：Hook 配置文件标注
- **当** 系统生成 `settings.json` 或 `hooks.json`
- **预期** 配置文件必须包含 `_generated_by: "@hunterzheng/harness"` 和 `_generated_at: "<ISO timestamp>"` 字段，便于后续 `--repair-adapters` 识别和更新

#### 需求项：事件写入路径统一

系统必须统一事件写入到 `.harness/events/` 路径，不再使用旧的 `skywalk-sdd/` 路径。

##### 场景：新事件路径
- **当** session-summary Hook 触发事件记录
- **预期** 事件数据必须写入 `.harness/events/<timestamp>-<event>.json`，不得写入 `skywalk-sdd/` 或其它旧路径

##### 场景：旧事件兼容只读
- **当** 系统读取历史事件
- **预期** 系统可以只读方式读取 `skywalk-sdd/` 中的旧事件，但新事件只写入 `.harness/events/`

### 修改需求

#### 需求项：dangerous-command 阻断集成到 CLI 流程

系统必须将 dangerous-command 阻断集成到 CLI 流程，在 `src/cli/main.ts` 命令执行前拦截危险命令。

**增量修改**：M1 阶段阻断集成范围为 CLI 命令级。AI 工具 Hook 侧的完整 dispatcher 和事件路由标记为后续版本。

##### 场景：CLI 执行前拦截（无变更）
- **当** 用户执行 `harness` 命令
- **预期** 阻断行为不变；但若通过 AI 工具原生 Hook（非 CLI 命令级）触发，标注为"AI 工具 Hook dispatcher 后续版本"并降级为 warning

#### 需求项：Subagent 定义文件生成

系统必须生成 Subagent 定义文件：需求分析 4 个、设计 4 个、代码生成 4 个、review 7 个。

**增量修改**：M1 阶段生成的 agent 定义文件标记为模板；Agent 的实际编排和调度（并行/串行/冲突解决）标记为后续版本。

##### 场景：生成需求分析 agent（补充约束）
- **当** 用户选择安装 Subagent
- **预期** 系统生成 agent 定义文件，每个文件头部必须包含 `_status: "template"` 标注，说明该 agent 为模板定义，实际 agent 编排和调度后续版本支持

### 移除需求

无。
