## 6. 模块内部逻辑

### 6.1 核心流程

```
resolveWorkspacePaths(root):
  harness = resolve(root, '.harness')
  return {
    root, harness,
    config: resolve(harness, 'config'),
    state: resolve(harness, 'state'),
    facts: resolve(harness, 'facts'),
    generated: resolve(harness, 'generated'),
    adapters: resolve(harness, 'adapters'),
    reports: resolve(harness, 'reports'),
    cache: resolve(harness, 'cache'),
    develop: resolve(harness, 'develop'),
    docs: resolve(harness, 'docs'),       // 新增
    rules: resolve(harness, 'rules'),     // 新增
    events: resolve(harness, 'events'),   // 新增
  }
```

### 6.2 幂等性保证

```
stageMkdir(tx, dir):
  if existsSync(dir): skip  // 幂等：已存在则跳过
  else: tx.operations.push({ type: 'create_dir', path: dir })
```

## 8. 异常处理

### 8.1 异常分类

| 异常类型 | 触发条件 | 处理策略 | 用户感知 |
|---------|---------|---------|---------|
| 目录创建失败 | 权限不足或磁盘满 | 返回错误码 2105 | "目录创建失败: {path}" |
| 配置字段缺失 | `generatedBlockPrefix` 等不存在 | 返回错误码 2106 | "配置缺少必填字段: {field}" |
| 配置文件写入失败 | review.config.json 写入失败 | 返回错误码 2107 | "配置文件写入失败" |

### 8.2 重试与降级

- 重试次数：0
- 降级策略：目录创建失败时通过 transaction 回滚已创建的目录
